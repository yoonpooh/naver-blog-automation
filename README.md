# 네이버 블로그 자동화

Codex Desktop에서 네이버 블로그용 키워드 조사부터 실제 발행까지 진행하는 개인용 자동화 스킬 모음입니다.

## 동작 환경

이 프로젝트는 다음 환경을 기준으로 만들었습니다.

- macOS
- Codex Desktop 앱
- 로그인된 Chrome의 네이버 블로그 편집기
- Codex의 Chrome 제어, Computer Use, 이미지 생성 기능

네이버 편집기와 macOS 네이티브 파일 선택기를 직접 조작하므로 일반 CLI, 헤드리스 브라우저, Windows, Linux에서는 정상 동작을 보장하지 않습니다. 현재로서는 **macOS의 Codex Desktop 앱 전용 워크플로**로 보는 편이 안전합니다.

## 자동화 순서

1. `keyword-research`: 급상승 후보와 꾸준한 저경쟁 후보를 조사해 키워드 하나를 선정합니다.
2. `title-generation`: 선정한 키워드로 제목 하나를 만듭니다.
3. `article-generation`: 공식 자료를 확인해 한국어 정보성 본문과 해시태그를 작성합니다.
4. `image-generation`: 본문의 모든 이미지 영역에 맞는 이미지를 병렬 생성합니다.
5. `article-publishing`: 로그인된 네이버 블로그 편집기에 제목, 본문, 이미지와 서식을 입력하고 검증 후 발행합니다.

## 준비

네이버 검색광고 키워드 도구를 사용하려면 `.env.example`을 `.env`로 복사하고 아래 값을 입력합니다.

```dotenv
NAVER_SEARCHAD_API_KEY=
NAVER_SEARCHAD_SECRET_KEY=
NAVER_SEARCHAD_CUSTOMER_ID=
```

그다음 Codex Desktop에서 이 저장소를 열고 다섯 스킬을 위 순서대로 실행하도록 요청하면 됩니다. 네이버 로그인이나 macOS 권한 승인이 필요할 때만 사용자 조작이 필요합니다.

## 발행 안전장치

- 이미지 업로드는 `사진` 클릭부터 네이티브 파일 선택기의 `열기`까지 Computer Use만 사용합니다.
- 이미지와 본문은 고유 마커와 컴포넌트 ID를 기준으로 배치합니다.
- 소제목 서식, 이미지 순서, AI 활용 설정, 해시태그와 카테고리를 발행 전에 검증합니다.
- 최종 발행 버튼은 한 번만 누르고, 게시글 화면의 제목과 URL이 확인돼야 성공으로 처리합니다.
- 생성된 글, 이미지와 업로드용 파일은 `output/`에 저장되며 Git에는 포함하지 않습니다.

## 참고

네이버 편집기 UI나 Codex Desktop의 브라우저 제어 방식이 바뀌면 스킬의 선택자와 작업 순서를 조정해야 할 수 있습니다. 실제 발행 전에는 로그인된 계정과 공개 범위를 확인하세요.
